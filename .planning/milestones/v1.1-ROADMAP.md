# Milestone v1.1: More Control

**Status:** In Progress
**Phases:** 6-10
**Total Plans:** TBD (pending breakdown)

## Overview

Give users full control over filtering, display formatting, and app behavior through Settings. Build on the existing v1.0 foundation with new SwiftData models for configurable ignored apps and patterns, a RedactionEngine for display formatting, and expanded Settings tabs for all user preferences.

## Phases

### Phase 6: Filtering

**Goal**: User can configure which apps and content patterns to ignore during clipboard capture
**Depends on**: Phase 5 (Settings Window exists)
**Requirements**: FILT-01, FILT-02, FILT-03

**Success Criteria:**
1. User can view and edit ignored app patterns in Settings (add, remove, rename)
2. User can add content ignore patterns using regex syntax
3. User can enable/disable individual patterns without deleting them
4. Clipboard content from ignored apps is not captured
5. Clipboard content matching ignore patterns is not stored

Plans:
- [x] 06-01-PLAN.md — Data models + ClipboardMonitor filtering
- [x] 06-02-PLAN.md — Settings UI for filtering

### Phase 7: Display

**Goal**: User can control how clipboard items appear in the menu preview
**Depends on**: Phase 6 (Settings infrastructure exists)
**Requirements**: DISP-01, DISP-02, DISP-03

**Success Criteria:**
1. User can configure preview truncation length in Settings
2. Invisible characters (newlines, tabs) display as symbols or are removed in preview
3. Sensitive content (emails, passwords) appears redacted in preview (e.g., `j***@e***.com`)
4. Full content is preserved in storage (redaction is display-only)

Plans:
- [x] 07-01-PLAN.md — RedactionPattern model + DisplayFormatter service
- [x] 07-02-PLAN.md — Display settings UI + wire to HistoryItemRow

### Phase 8: App Behavior

**Goal**: User can configure app-wide behavior including startup, limits, and hotkeys
**Depends on**: Phase 7 (Settings tabs pattern established)
**Requirements**: BEHV-01, BEHV-02, BEHV-03, BEHV-04

**Success Criteria:**
1. User can enable/disable "Launch at Login" in Settings
2. User can set maximum history items (and existing items are pruned to match)
3. User can customize the global hotkey via keyboard recorder
4. User can configure auto-cleanup age (items older than X days are deleted automatically)

Plans:
- [x] 08-01-PLAN.md — LaunchAtLogin dependency, dynamic maxItems, auto-cleanup timer, GeneralSettingsView with all 4 controls

### Phase 9: Settings Navigation

**Goal**: Replace flat settings layout with tab-like navigation for better organization
**Depends on**: Phase 8 (All settings tabs implemented)
**Requirements**: NAV-01

**Success Criteria:**
1. Settings window uses tab-like navigation (sidebar or segmented control)
2. Each settings category (General, Filtering, Display, Transforms, Hooks) is a separate navigable section
3. Navigation feels native to macOS settings conventions

## Technical Notes

**New Dependencies:**
- LaunchAtLogin-Modern (v1.1.0) — for BEHV-01

**Key Risks (from research):**
1. SwiftData schema migration — adding new models to existing container
2. Regex performance — cache compiled patterns
3. LaunchAtLogin App Store guidelines — never auto-enable

**Architecture Impact:**
- New models: IgnoredApp, IgnoredPattern
- New service: RedactionEngine
- Modified: ClipboardMonitor (filtering), HistoryItemRow (display), SettingsView (tabs)
