# Milestone v1.0: MVP

**Status:** SHIPPED 2026-02-05
**Phases:** 1-5
**Total Plans:** 9

## Overview

Build a macOS menu bar clipboard manager with intelligent transforms. Start with the foundation (menu bar shell + clipboard monitoring), add persistent history with search, implement the transform rule engine, then wire up external hooks and automations.

## Phases

### Phase 1: Foundation

**Goal**: Working menu bar app that monitors clipboard and captures text content
**Depends on**: Nothing (first phase)
**Plans**: 2 plans

Plans:
- [x] 01-01: App skeleton with menu bar icon
- [x] 01-02: Clipboard monitoring and global hotkey

**Key deliverables:**
- MenuBarExtra with .window style for custom popup
- ClipboardMonitor service with 500ms polling
- Global hotkey Cmd+Shift+V
- Password manager content filtering (TransientType/ConcealedType/AutoGeneratedType)

### Phase 2: History

**Goal**: Persistent clipboard history with searchable popup UI
**Depends on**: Phase 1
**Plans**: 2 plans

Plans:
- [x] 02-01: History storage and popup UI
- [x] 02-02: Search and quick paste

**Key deliverables:**
- SwiftData persistence (SQLite backend)
- Max 100 items with automatic pruning
- Case-insensitive search filtering
- Click-to-copy with hover feedback
- Item deletion via context menu

### Phase 3: Transforms

**Goal**: Rule engine that auto-transforms clipboard content based on patterns or source app
**Depends on**: Phase 2
**Plans**: 2 plans

Plans:
- [x] 03-01: Transform rule engine
- [x] 03-02: Rules UI and default rules

**Key deliverables:**
- TransformRule SwiftData model for regex rules
- TransformEngine applies rules and updates clipboard
- Default rules: strip trailing whitespace, normalize line endings
- RulesView/RuleEditorView with live regex testing
- Reset to Defaults functionality

### Phase 4: Hooks

**Goal**: External triggers that fire scripts/apps on clipboard events
**Depends on**: Phase 3
**Plans**: 2 plans

Plans:
- [x] 04-01: Hook system and external triggers
- [x] 04-02: Hooks UI

**Key deliverables:**
- Hook SwiftData model for external command automation
- HookEngine service with fire-and-forget async execution
- Environment variables CLIPASS_CONTENT and CLIPASS_SOURCE_APP
- HooksView/HookEditorView for full CRUD UI

### Phase 5: Settings Window

**Goal**: Move rules and hooks configuration from popup to a dedicated Settings window
**Depends on**: Phase 4
**Plans**: 1 plan

Plans:
- [x] 05-01: Settings window with rules and hooks tabs

**Key deliverables:**
- SettingsView with TabView for Rules/Hooks
- Simplified popup showing only clipboard history
- Proper Window scene (appears in app switcher)

**Key Decisions:**
- Text-only v1 (simplifies transform logic)
- Local-only v1 (sync deferred to future)
- Swift/SwiftUI for native macOS experience
- 500ms polling for clipboard monitoring
- SwiftData for persistence
- MenuBarExtra with .window style

**Issues Resolved:**
- Monitor starts on app launch (moved init to App struct)
- Copy-from-history no longer creates duplicates (content comparison)

- None

**Technical Debt Incurred:**
- None significant
