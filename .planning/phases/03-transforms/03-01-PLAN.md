<task type="auto">
  <name>Task 2: Create TransformEngine and integrate with ClipboardMonitor</name>
  <files>clipass/Services/TransformEngine.swift, clipass/Services/ClipboardMonitor.swift</files>
  <action>
1. Create clipass/Services/TransformEngine.swift:
   - Class with modelContext injection (like ClipboardMonitor)
   - Method: `transform(_ content: String, sourceApp: String?) -> String`
     - Fetch all enabled TransformRules sorted by order
     - Filter by sourceAppFilter (nil = applies to all, or matches sourceApp)
     - For each rule, try to compile pattern as Regex
     - Apply replacement using `content.replacing(regex, with: replacement)`
     - Return transformed content
   - Handle invalid regex gracefully (skip rule, log warning)

2. Update ClipboardMonitor to use TransformEngine:
   - Add transformEngine property (set via setTransformEngine method)
   - In poll(), after getting content and sourceApp:
     - Call transformEngine.transform(content, sourceApp: sourceApp)
     - Use transformed content for the ClipboardItem
   - Also update the clipboard itself with transformed content:
     - NSPasteboard.general.clearContents()
     - NSPasteboard.general.setString(transformedContent, forType: .string)
     - Update lastChangeCount to avoid re-polling our own change

**Avoid:**
- Blocking main thread with regex operations — transforms run on same utility queue as poll()
- Infinite loop when updating clipboard — must update lastChangeCount after setString
  </action>
  <verify>swift build succeeds, manually test by adding a rule via SwiftData and copying text</verify>
  <done>TransformEngine applies rules to content, ClipboardMonitor uses transformed content, clipboard is updated with transformed text, no infinite loops</done>
</task>
