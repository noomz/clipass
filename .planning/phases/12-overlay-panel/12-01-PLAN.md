Create the NSPanel foundation for the floating overlay panel with hotkey-driven show/hide/toggle, ESC dismissal, and click-outside dismissal.

Purpose: This is the window-level infrastructure that all overlay UI (Phase 12 Plan 02) builds on top of. Without a properly configured non-activating NSPanel, no overlay feature can work.

<task type="auto">
  <name>Task 1: Create OverlayPanel and OverlayWindowController</name>
  <files>clipass/Controllers/OverlayWindowController.swift</files>
  <action>
Create a new `Controllers/` directory and `OverlayWindowController.swift` containing two types:

**OverlayPanel (NSPanel subclass):**
- Init with `contentRect: NSRect(x: 0, y: 0, width: 640, height: 400)`, styleMask: `[.nonactivatingPanel, .titled, .fullSizeContentView]`, backing: `.buffered`, defer: `false`
- Set in init: `isFloatingPanel = true`, `level = .floating`, `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]`
- Hide titlebar: `titleVisibility = .hidden`, `titlebarAppearsTransparent = true`, hide all three standard window buttons
- Set: `isMovableByWindowBackground = true`, `isOpaque = false`, `backgroundColor = .clear`, `hasShadow = true`
- Override `canBecomeKey` and `canBecomeMain` to return `true` (CRITICAL: NSPanel defaults these to false)
- Override `resignKey()`: call `super.resignKey()`, then check a guard `guard Date().timeIntervalSince(shownAt) > 0.3 else { return }` to prevent immediate dismissal (see Pitfall 3 in RESEARCH.md), then call `orderOut(nil)` and post `Notification.Name("overlayDidResignKey")` so the controller can restore focus
- Add a `var shownAt: Date = .distantPast` property that `OverlayWindowController.show()` sets

**OverlayWindowController (@MainActor singleton):**
- `static let shared = OverlayWindowController()`
- Private `panel: OverlayPanel` property
- Private `previousApp: NSRunningApplication?` property
- `private init()`: create `OverlayPanel`, set `panel.contentView = NSHostingView(rootView: Text("Overlay placeholder").frame(width: 640, height: 400))` as temporary placeholder content. Observe the `overlayDidResignKey` notification to call `restoreFocus()`.
- `func toggle()`: if `panel.isVisible` call `hide()`, else call `show()`
- `func show()`: capture `previousApp = NSWorkspace.shared.frontmostApplication`, position panel centered on the screen containing the mouse cursor (slightly above vertical center, see RESEARCH.md code example), set `panel.shownAt = Date()`, call `panel.makeKeyAndOrderFront(nil)`, add delayed fallback `DispatchQueue.main.asyncAfter(deadline: .now() + 0.1)` to call `panel.makeFirstResponder(panel.contentView)` for focus
- `func hide()`: call `panel.orderOut(nil)`, call `restoreFocus()`
- `private func restoreFocus()`: call `previousApp?.activate(options: .activateIgnoringOtherApps)`, set `previousApp = nil`
- `func pasteAndHide(content: String)`: write content to `NSPasteboard.general` (clearContents + setString), then call `hide()`
- Add a `func setContentView<V: View>(_ view: V)` method that replaces `panel.contentView` with a new `NSHostingView` wrapping the provided view (Plan 02 will call this to inject the real overlay UI)

IMPORTANT: Do NOT call `NSApp.activate()` anywhere in this file. The panel is non-activating by design.
