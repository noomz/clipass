Add inline editing to the overlay panel. Users click a pencil icon on a clipboard row to open a bottom editor panel with the raw content in a monospace NSTextView. Edited content syncs to the menu bar popup automatically via @Query.

**1. Modify `clipass/Views/ClipboardOverlayView.swift`** — Add edit state and wire everything:

@State private var editingItemID: ClipboardItem.ID? = nil
@State private var editorContent: String = ""

Add helper functions:
- `enterEditMode(for item: ClipboardItem)`: Sets `editorContent = item.content` (raw, NOT DisplayFormatter output), then `withAnimation(.easeOut(duration: 0.15)) { editingItemID = item.id }`
- `commitEdit()`: Finds item by editingItemID in filteredItems, sets `item.content = editorContent`, calls `try? modelContext.save()`, then `withAnimation(.easeOut(duration: 0.15)) { editingItemID = nil }`. If item not found, calls cancelEdit(). Does NOT write to NSPasteboard.
- `cancelEdit()`: Sets `editorContent = ""`, `withAnimation(.easeOut(duration: 0.15)) { editingItemID = nil }`

Modify OverlaySearchField construction in contentStack:
- `onArrowUp`: `editingItemID == nil ? moveSelectionUp : {}`  (no-op closures, not nil)
- `onArrowDown`: `editingItemID == nil ? moveSelectionDown : {}`
- `onEscape`: `{ if editingItemID != nil { cancelEdit() } else { OverlayWindowController.shared.hide() } }`
- `onReturn`: `editingItemID == nil ? pasteSelected : {}`
- Add `.disabled(editingItemID != nil)` and `.opacity(editingItemID != nil ? 0.5 : 1.0)` to the search field

Add `shouldRefocus` parameter support: pass `shouldRefocus: editingItemID == nil` to OverlaySearchField (see OverlaySearchField changes below)

Modify OverlayItemRow construction in the ForEach:
- Add `onEdit: { enterEditMode(for: item) }`
- Add `isEditing: item.id == editingItemID`

Replace the bottom section of contentStack (after ScrollViewReader): Instead of always showing themedDivider + bottom bar, use conditional:
- If `editingItemID != nil` and matching item found: show themedDivider + `InlineEditorPanel(content: $editorContent, theme: theme, onSave: commitEdit, onCancel: cancelEdit).frame(height: 160).transition(.move(edge: .bottom).combined(with: .opacity))`
- Else: show existing themedDivider + bottom bar (item count)

Wrap the contentStack body in `animation(.easeOut(duration: 0.15), value: editingItemID)` or use explicit `withAnimation` in enter/commit/cancel functions (already specified above).

Reset edit state in overlayWillShow notification handler: add `editingItemID = nil` and `editorContent = ""`

- In `updateNSView`, after the existing logic: if `shouldRefocus` is true and the field is NOT currently first-responder, call `DispatchQueue.main.async { nsView.window?.makeFirstResponder(nsView) }`. This ensures focus returns to the search field after editing ends. Check via `nsView.window?.firstResponder == nsView.currentEditor() || nsView.window?.firstResponder == nsView`

**3. Modify `clipass/Controllers/OverlayWindowController.swift`** — Guard cancelOperation for edit state:

- Add a property to OverlayPanel: `var cancelEditHandler: (() -> Bool)? = nil` — returns true if edit was cancelled, false if no edit active
- In `OverlayPanel.cancelOperation(_:)`: call `if cancelEditHandler?() == true { return }` before the existing `OverlayWindowController.shared.hide()` call
- In ClipboardOverlayView, add an `.onAppear` or use the existing `.onAppear` to register the handler: access the panel via `OverlayWindowController.shared` and set the closure. The closure checks `editingItemID != nil`, calls `cancelEdit()`, and returns true. If `editingItemID == nil`, returns false (letting hide() proceed).
- The simplest approach: In ClipboardOverlayView, add `.onAppear { if let panel = NSApp.windows.compactMap({ $0 as? OverlayPanel }).first { panel.cancelEditHandler = { [self] in ... } } }`. Alternatively, expose panel on OverlayWindowController: add `var panel: OverlayPanel { overlayPanel }` accessor (if overlayPanel is private, make it internal or add accessor). Then in ClipboardOverlayView's onAppear or onChange of editingItemID, set `OverlayWindowController.shared.panel.cancelEditHandler`.
- Research Pitfall 7 documents why this is needed: if EditorTextView loses first-responder and user presses ESC, panel-level cancelOperation fires and bypasses two-stage ESC.
