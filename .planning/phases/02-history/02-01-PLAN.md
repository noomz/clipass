<objective>
Add persistent storage for clipboard history and improve the popup UI to show full scrollable history.

Purpose: Without persistence, history is lost on app restart. Users need their clipboard history to survive restarts and have enough items to be useful.
Output: SwiftData-backed history that persists across launches, scrollable popup showing all items with timestamps.
</objective>

  <name>Task 1: Add SwiftData persistence to ClipboardItem</name>
  <files>clipass/Models/ClipboardItem.swift, clipass/clipassApp.swift</files>
  <action>
1. Update ClipboardItem to be a SwiftData model:
   - Add `import SwiftData`
   - Change `struct ClipboardItem` to `@Model class ClipboardItem`
   - Keep existing properties: id (UUID), content (String), sourceApp (String?), timestamp (Date)
   - Remove `Identifiable` conformance (SwiftData models are automatically Identifiable)
   - Keep the convenience init

2. Update clipassApp.swift:
   - Add `import SwiftData`
   - Add `.modelContainer(for: ClipboardItem.self)` modifier to the MenuBarExtra scene
   - This creates the SQLite database automatically

**Avoid:** Using Core Data directly — SwiftData is the modern replacement with simpler API.
  </action>
  <verify>swift build succeeds, app launches without SwiftData errors</verify>
  <done>ClipboardItem is a @Model class, modelContainer configured in app, no compile errors</done>
</task>

<task type="auto">
  <name>Task 2: Wire ClipboardMonitor to SwiftData and update popup UI</name>
  <files>clipass/Services/ClipboardMonitor.swift, clipass/Views/ClipboardPopup.swift</files>
  <action>
1. Update ClipboardMonitor to save items to SwiftData:
   - Add modelContext property (injected or passed from view)
   - In poll(), after creating ClipboardItem, insert into modelContext
   - Remove the in-memory history array (SwiftData is the source of truth)
   - Add a configurable maxItems limit (default 100) — delete oldest when exceeded

2. Update ClipboardPopup to query SwiftData directly:
   - Use @Query to fetch ClipboardItem sorted by timestamp descending
   - Remove dependency on ClipboardMonitor.history array
   - Show ALL items in ScrollView (not just last 5)
   - Display timestamp in relative format ("2m ago", "1h ago")
   - Show item count in header

3. Update how ClipboardMonitor gets modelContext:
   - Option A: Pass modelContext from view via @Environment
   - Option B: ClipboardMonitor holds a reference passed at init
   - Use Option B — ClipboardMonitor is created in app, pass context at creation

**Avoid:**
- Keeping both in-memory array AND SwiftData (redundant, can get out of sync)
- Fetching all history on every poll (let SwiftUI's @Query handle reactivity)
  </action>
