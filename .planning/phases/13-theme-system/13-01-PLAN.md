  <files>clipass/Models/Theme.swift, clipass/Services/ThemeManager.swift</files>
  <action>
Create `clipass/Models/Theme.swift` containing:

1. `Color` extension with `init(hex: String)` initializer (Scanner + UInt64 approach from research).

2. Nord palette color constants as static properties on `Color`:
   - Polar Night: nord0 `#2e3440`, nord1 `#3b4252`, nord2 `#434c5e`, nord3 `#4c566a`
   - Snow Storm: nord4 `#d8dee9`, nord5 `#e5e9f0`, nord6 `#eceff4`
   - Frost: nord7 `#8fbcbb`, nord8 `#88c0d0`, nord9 `#81a1c1`, nord10 `#5e81ac`
   - Aurora: nord11 `#bf616a`, nord14 `#a3be8c`, nord15 `#b48ead`

3. `enum BackgroundMode` with cases:
   - `.vibrancy(material: NSVisualEffectView.Material)`
   - `.tintedVibrancy(material: NSVisualEffectView.Material, tint: Color, opacity: Double)`
   - `.solid`

4. `enum DividerStyle` with cases: `.standard`, `.thick`, `.none`

5. `enum ThemeID: String, CaseIterable` with cases: `system`, `dark`, `light`, `midnight`, `nord`

6. `struct Theme` with properties exactly as specified in research Pattern 1:
   - `id: ThemeID`, `name: String`
   - Colors: `overlayBackground`, `itemBackground`, `primaryText`, `secondaryText`, `dividerColor`, `searchFieldBackground`, `searchFieldText`, `searchFieldPlaceholder`, `accentColor` (all `Color`)
   - Structure: `cornerRadius: CGFloat`, `itemCornerRadius: CGFloat`, `itemVerticalPadding: CGFloat`, `dividerStyle: DividerStyle`, `titleFontWeight: Font.Weight`, `bodyFontSize: CGFloat`
   - `backgroundMode: BackgroundMode`
   - Add `forceAppearance: NSAppearance?` property for fixed dark/light vibrancy themes

7. `static let themes: [ThemeID: Theme]` dictionary with all 5 theme definitions using the exact palettes from research code examples. Per user decision, each theme must have a strong distinct personality:
   - **System**: vibrancy with `.hudWindow` material, tinted with `Color.accentColor` at opacity 0.15, `forceAppearance: nil` (follows system). Use system colors (.primary, .secondary, Color.accentColor).
   - **Dark**: solid background `#1e1e1e`, blue accent `#3a7bd5`, `forceAppearance: nil` (solid mode, no vibrancy).
   - **Light**: vibrancy with `.hudWindow` material, forced `.aqua` appearance. Light palette per research.
   - **Midnight**: tinted vibrancy with purple tint `#2d0a5e` at opacity 0.6, forced `.darkAqua`. Purple/violet palette per research. Sharper corners (10), `.thick` dividers, `.medium` font weight.
   - **Nord**: solid background using nordPolarNight0. Official Nord hex values. Softer corners (14). `bodyFontSize: NSFont.systemFontSize - 0.5`.

Create `clipass/Services/ThemeManager.swift` containing:

1. `@Observable final class ThemeManager` with:
   - `var current: Theme` — the active theme, initialized from stored preference
   - `@ObservationIgnored @AppStorage("selectedThemeID") private var storedThemeID: String = ThemeID.system.rawValue`
   - `var selectedID: ThemeID` — computed property returning `ThemeID(rawValue: storedThemeID) ?? .system`
   - `init()` — reads `storedThemeID`, sets `current` from `Theme.themes[selectedID]`
   - `func select(_ id: ThemeID)` — sets `storedThemeID = id.rawValue`, sets `current = Theme.themes[id]!`

IMPORTANT: `@AppStorage` does NOT support ThemeID directly. Store the rawValue String and convert. Do NOT use Codable RawRepresentable bridging (infinite recursion trap per research Pitfall 6).

**Update `clipass/Views/VisualEffectView.swift`:**

Replace the current `material`/`blendingMode` interface with a `BackgroundMode`-driven approach:

1. Change init to accept `backgroundMode: BackgroundMode` and `forceAppearance: NSAppearance? = nil` instead of material/blendingMode.
2. `makeNSView`: Switch on backgroundMode:
   - `.vibrancy(material)`: Create NSVisualEffectView with material, `.behindWindow` blending, `state = .active` (CRITICAL for .accessory policy). Set `forceAppearance` if provided.
   - `.tintedVibrancy(material, tint, opacity)`: Same as vibrancy but the view is an NSView container. Add NSVisualEffectView as sublayer, then add a tint overlay view (NSView with wantsLayer, layer.backgroundColor set to the tint NSColor with specified opacity). Set `forceAppearance` on the visual effect view.
   - `.solid`: Return plain NSView with `wantsLayer = true`, `layer.backgroundColor` set from a separate `solidColor: NSColor?` parameter.
3. Add `solidColor: NSColor? = nil` parameter for solid mode background.
4. `updateNSView`: Re-apply material, state = .active (CRITICAL — must be set on every update per Pitfall 5), forceAppearance, and tint/solid colors. Set `state = .active` in updateNSView alongside any material update.

Also keep backward compatibility: provide a convenience init that takes `material` and `blendingMode` and maps to `.vibrancy(material:)` — BUT actually, ClipboardOverlayView will be updated in Plan 02 to use the new interface, so just update the signature directly. Plan 02 will update ClipboardOverlayView to pass the theme's backgroundMode.
