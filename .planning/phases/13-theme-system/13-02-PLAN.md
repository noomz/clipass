**Update `clipass/Views/ClipboardOverlayView.swift`:**

1. Add `@Environment(ThemeManager.self) private var themeManager` at the top of the struct.
2. Create a computed `var theme: Theme { themeManager.current }` convenience property.
3. Replace `VisualEffectView()` with `VisualEffectView(backgroundMode: theme.backgroundMode, forceAppearance: theme.forceAppearance, solidColor: NSColor(theme.overlayBackground))`.
4. Replace `.clipShape(RoundedRectangle(cornerRadius: 12))` with `.clipShape(RoundedRectangle(cornerRadius: theme.cornerRadius))`.
5. Replace `Divider()` calls with themed dividers based on `theme.dividerStyle`:
   - For `.standard`: `Rectangle().fill(theme.dividerColor).frame(height: 1)`
   - For `.thick`: `Rectangle().fill(theme.dividerColor).frame(height: 2)`
   - For `.none`: `EmptyView()`
   Create a `@ViewBuilder private var themedDivider: some View` computed property for this.
6. Update bottom bar text: replace `.foregroundColor(.secondary)` with `.foregroundColor(theme.secondaryText)`.
7. Pass `theme` to `OverlaySearchField` — add a `theme: Theme` parameter to OverlaySearchField so it can apply colors in `updateNSView`.

**Update `clipass/Views/OverlayItemRow.swift`:**

1. Add `@Environment(ThemeManager.self) private var themeManager` and convenience `var theme`.
2. Replace ALL hardcoded colors:
   - Selected text: `.white` -> keep `.white` (correct for all 5 themes since itemBackground is always a saturated/dark color)
   - Non-selected primary text: `.primary` -> `theme.primaryText`
   - Non-selected secondary text: `.secondary` -> `theme.secondaryText`
   - Selected secondary text: `.white.opacity(0.75)` -> keep `Color.white.opacity(0.75)` (works for all dark item backgrounds)
   - Selection highlight: `Color.accentColor` -> `theme.itemBackground`
   - Hover background: `Color.primary.opacity(0.08)` -> `theme.primaryText.opacity(0.08)`
   - Corner radius: hardcoded `6` -> `theme.itemCornerRadius`
   - Vertical padding: hardcoded `6` -> `theme.itemVerticalPadding`
   - Font: `.body` -> `.system(size: theme.bodyFontSize, weight: theme.titleFontWeight)` for the content text
   - Pin icon color: `.orange` stays orange (brand color, not themed)

**Update `clipass/Views/OverlaySearchField.swift`:**

1. Add a `theme: Theme` parameter to `OverlaySearchField` struct (not via @Environment — NSViewRepresentable cannot reliably use @Environment for NSView updates; pass explicitly).
2. In `updateNSView`:
   - Set `nsView.textColor = NSColor(theme.searchFieldText)`
   - Set placeholder color via attributed string: `nsView.placeholderAttributedString = NSAttributedString(string: placeholder, attributes: [.foregroundColor: NSColor(theme.searchFieldPlaceholder), .font: nsView.font ?? NSFont.systemFont(ofSize: NSFont.systemFontSize)])`
   - For search field background: check `theme.backgroundMode`. If `.solid`, set `nsView.drawsBackground = true` and `nsView.backgroundColor = NSColor(theme.searchFieldBackground)`. Otherwise set `nsView.drawsBackground = false`.
3. Also apply in `makeNSView` for initial state (same logic).
4. In `updateNSView`, also update font size: `nsView.font = NSFont.systemFont(ofSize: theme.bodyFontSize)`.

   - Background: solid rectangle filled with `theme.overlayBackground` (no actual vibrancy in the miniature — vibrancy requires NSVisualEffectView which does not scale well in SwiftUI previews).
   - Search field area: rounded rectangle with `theme.searchFieldBackground`, placeholder text "Search clipboard..." in `theme.searchFieldPlaceholder`.
   - Themed divider below search.
   - 3 fake clipboard items as rows:

     Each row shows a secondary text line (e.g., "Safari - 2m ago") in `theme.secondaryText`.
   - Themed divider above bottom bar.
   - Bottom bar: "3 items" in `theme.secondaryText`.
   - Use `theme.cornerRadius` for outer clip, `theme.itemCornerRadius` for item backgrounds.
   - Use `theme.bodyFontSize` and `theme.titleFontWeight` for text.

2. Create `ThemePreviewCard` view:
   - Takes `theme: Theme` and `isSelected: Bool`.
   - Renders `MiniOverlayMockup(theme: theme)` at full resolution `.frame(width: 480, height: 300)` then `.scaleEffect(0.5, anchor: .topLeading)` and `.frame(width: 240, height: 150)` to collapse layout (CRITICAL per research Pitfall 2 — scaleEffect does NOT change layout size).
   - Below the mockup: theme name label in `.headline` weight, left-aligned.
   - If selected: checkmark badge (SF Symbol `checkmark.circle.fill`) overlaid at top-trailing corner. Border stroke in `theme.accentColor` at 2pt width.
   - If not selected: subtle border at 1pt in `.primary.opacity(0.15)`.
   - Outer corner radius: 10.
   - Entire card is clipped with `RoundedRectangle(cornerRadius: 10)`.

3. Create `AppearanceSettingsView`:
   - `@Environment(ThemeManager.self) private var themeManager`
   - `ScrollView` containing `VStack(spacing: 16)` of `ThemePreviewCard` for each `ThemeID.allCases`.
   - On tap of a card: `themeManager.select(id)` — instant apply per user decision.
   - Padding around the VStack.
