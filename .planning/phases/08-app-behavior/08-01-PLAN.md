Add a periodic auto-cleanup timer to the `AppServices` class that deletes ClipboardItems older than the configured age.

In `AppServices`, add two methods:

1. **`startAutoCleanupTimer()`** — Called from `initialize()` after `clipboardMonitor.start()`. Creates a repeating Timer that fires every 3600 seconds (1 hour). On each fire, dispatches to main queue and calls `performAutoCleanup()`. Also run cleanup once immediately on startup.

```swift
private func startAutoCleanupTimer() {
    performAutoCleanup()
    Timer.scheduledTimer(withTimeInterval: 3600, repeats: true) { [weak self] _ in
        DispatchQueue.main.async {
            self?.performAutoCleanup()
        }
    }
}
```

2. **`performAutoCleanup()`** — Reads `UserDefaults.standard.integer(forKey: "autoCleanupDays")`. If 0, does nothing (disabled). Otherwise, computes cutoff date (`Calendar.current.date(byAdding: .day, value: -days, to: Date())`), fetches ClipboardItems with `timestamp < cutoffDate` using `#Predicate`, deletes them, and saves context.

```swift
private func performAutoCleanup() {
    let days = UserDefaults.standard.integer(forKey: "autoCleanupDays")
    guard days > 0 else { return }
    
    guard let cutoffDate = Calendar.current.date(byAdding: .day, value: -days, to: Date()) else { return }
    let context = modelContainer.mainContext
    let predicate = #Predicate<ClipboardItem> { $0.timestamp < cutoffDate }
    let descriptor = FetchDescriptor<ClipboardItem>(predicate: predicate)
    
    guard let oldItems = try? context.fetch(descriptor) else { return }
    for item in oldItems {
        context.delete(item)
    }
    if !oldItems.isEmpty {
        try? context.save()
    }
}
```

Add the `startAutoCleanupTimer()` call in `initialize()`, right after `clipboardMonitor.start()`:

```swift

```

Important: The timer fires on the main run loop (since `AppServices.initialize()` runs on main actor). The `DispatchQueue.main.async` in the timer callback ensures SwiftData context access is on the main actor. Use `UserDefaults.standard` directly (not `@AppStorage`) since this is not a SwiftUI view.

New `GeneralSettingsView` implementation:

```swift
struct GeneralSettingsView: View {
    @AppStorage("maxHistoryItems") private var maxHistoryItems = 100
    @AppStorage("autoCleanupDays") private var autoCleanupDays = 0
    @Environment(\.modelContext) private var modelContext

    var body: some View {
        Form {
            // BEHV-01: Launch at Login
            Section {
                LaunchAtLogin.Toggle()
            } header: {
                Text("Startup")
            }

            // BEHV-02: Max history items
            Section {
                Stepper("Maximum items: \(maxHistoryItems)",
                        value: $maxHistoryItems,
                        in: 10...1000,
                        step: 10)
                    .onChange(of: maxHistoryItems) { _, newValue in
                        pruneToLimit(newValue)
                    }
            } header: {
                Text("History")
            }

            // BEHV-03: Global hotkey customization
            Section {
                KeyboardShortcuts.Recorder("Toggle Clipboard:", name: .toggleClipboard)
            } header: {
                Text("Hotkey")
            }

            // BEHV-04: Auto-cleanup age
            Section {
                Picker("Auto-delete items older than:", selection: $autoCleanupDays) {
                    Text("Never").tag(0)
                    Text("1 day").tag(1)
                    Text("7 days").tag(7)
                    Text("30 days").tag(30)
                    Text("90 days").tag(90)
                }
            } header: {
                Text("Cleanup")
            }
        }
        .formStyle(.grouped)
        .padding()
    }

    private func pruneToLimit(_ limit: Int) {
        let descriptor = FetchDescriptor<ClipboardItem>(
            sortBy: [SortDescriptor(\.timestamp, order: .reverse)]
        )
        guard let allItems = try? modelContext.fetch(descriptor) else { return }
        if allItems.count > limit {
            for item in allItems.suffix(from: limit) {
                modelContext.delete(item)
            }
            try? modelContext.save()
        }
    }
}
```

Key implementation details:
- **LaunchAtLogin.Toggle()** — Drop-in toggle that manages SMAppService state internally. Never set `LaunchAtLogin.isEnabled` programmatically. The toggle defaults to OFF (per Apple guidelines).
- **Stepper** — Range 10-1000, step 10. The `onChange` triggers immediate pruning when the value decreases, so excess items are deleted right away (not just on next clipboard event).
- **KeyboardShortcuts.Recorder** — Uses name-based init (`.toggleClipboard`), NOT binding-based. This ensures the global hotkey re-registers automatically when the user records a new shortcut. The existing `KeyboardShortcuts.onKeyUp(for: .toggleClipboard)` in clipassApp.swift picks up changes via UserDefaults KVO.
- **Picker for cleanup age** — Uses tag(0) for "Never" (disabled). The auto-cleanup timer in AppServices checks this value each hour and skips when 0.
- **@AppStorage keys** — "maxHistoryItems" matches ClipboardMonitor's UserDefaults key. "autoCleanupDays" matches AppServices' performAutoCleanup key.
