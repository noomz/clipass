# Plan 11-01: Smart Context Actions

## Goal

Add smart, content-aware context menu actions to clipboard history items, plus a user-customizable actions system managed in Settings.

## Design

### Content Detection (ContentAnalyzer)

A lightweight service that analyzes clipboard content and returns detected types:

| Type | Detection | Example |
|------|-----------|---------|
| `url` | Regex: `https?://...` or starts with `www.` | `https://example.com` |
| `email` | Regex: `name@domain.tld` | `user@gmail.com` |
| `json` | Try JSONSerialization parse | `{"key": "value"}` |
| `number` | Regex: numeric string | `42`, `3.14`, `-100` |
| `path` | Starts with `/` or `~/` | `/usr/local/bin` |
| `hex` | Matches `#[0-9a-fA-F]{3,8}` | `#FF5733` |
| `text` | Default fallback | Everything else |

### Built-in Actions

Always available:
- **Copy** (existing)
- **Delete** (existing)
- **Pin/Unpin** — toggle pinned state (pinned items exempt from auto-cleanup and max-items pruning)

Content-aware (shown only when detected):
- **Open URL** — when `url` detected, open in default browser
- **Send Email** — when `email` detected, open `mailto:` link
- **Format JSON** — when `json` detected, copy pretty-printed JSON
- **Open in Finder** — when `path` detected, reveal in Finder
- **Copy Uppercase / Copy Lowercase** — always available for text
- **Copy Trimmed** — always available, copies with whitespace stripped
- **Copy URL Decoded / Encoded** — when `url` detected
- **Copy Base64 Encoded / Decoded** — always available

### Custom Actions (ContextAction model)

SwiftData model for user-defined actions:
- `name: String` — display label
- `command: String` — shell command to execute (content piped via CLIPASS_CONTENT env var)
- `contentFilter: String` — regex pattern; action only shows when content matches (empty = always show)
- `replacesClipboard: Bool` — if true, stdout replaces clipboard content
- `isEnabled: Bool`
- `order: Int`

Custom actions appear in a "Custom Actions" submenu in the context menu.

### Settings Tab

Add "Actions" tab to SettingsView with:
- List of custom actions (add/edit/delete)
- Editor sheet for creating/editing actions (same pattern as HookEditorView)
- Toggle for built-in actions visibility (future, not this phase)

### Pin Support

Add `isPinned: Bool` to ClipboardItem model. Pinned items:
- Show a pin icon in the row
- Are exempt from auto-cleanup timer
- Are exempt from max-items pruning
- Sort to top of list (pinned first, then by timestamp)

## Tasks

### Task 1: ContentAnalyzer service
- Create `clipass/Services/ContentAnalyzer.swift`
- Implement `analyze(_ content: String) -> Set<ContentType>` 
- ContentType enum with cases: url, email, json, number, path, hexColor, text

### Task 2: Pin support on ClipboardItem
- Add `isPinned: Bool = false` to ClipboardItem model
- Register in ModelContainer (no migration needed — SwiftData handles new bool with default)
- Update ClipboardPopup query to sort pinned first: `isPinned` desc, then `timestamp` desc
- Update auto-cleanup in clipassApp to exclude pinned items
- Update max-items pruning in GeneralSettingsView to exclude pinned items
- Add pin icon to HistoryItemRow when pinned

### Task 3: ContextAction model + engine
- Create `clipass/Models/ContextAction.swift` — SwiftData model
- Register ContextAction in ModelContainer
- Create `clipass/Services/ContextActionEngine.swift` — runs shell command with CLIPASS_CONTENT env, optionally captures stdout to replace clipboard

### Task 4: Smart context menu in HistoryItemRow
- Refactor `.contextMenu` in HistoryItemRow to show smart actions
- Always: Copy, Pin/Unpin, Copy Uppercase, Copy Lowercase, Copy Trimmed, Copy Base64 Encode/Decode, Delete
- Conditional: Open URL, Send Email, Format JSON, Open in Finder, Copy URL Encoded/Decoded
- Custom actions submenu (fetched via @Query)
- Use ContentAnalyzer to determine which conditional actions to show

### Task 5: Settings UI for custom actions
- Create `clipass/Views/ContextActionsView.swift` — list with add/edit/delete
- Create `clipass/Views/ContextActionEditorView.swift` — editor sheet
- Add "Actions" tab to SettingsView

## Verification

- Build succeeds with `swift build`
- Right-click a URL item → "Open URL" appears
- Right-click a non-URL item → "Open URL" does NOT appear
- Pin an item → pin icon shows, item sorts to top
- Pinned items survive auto-cleanup
- Custom action executes shell command with content
- Actions tab appears in Settings with CRUD for custom actions
