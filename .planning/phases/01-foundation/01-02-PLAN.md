**Tech stack from research:**
- NSPasteboard.changeCount polling (500ms interval)
- KeyboardShortcuts library for global hotkey
- @Observable macro for ClipboardMonitor

**Don't hand-roll:**
- Global hotkey registration — use KeyboardShortcuts library
- Clipboard change detection — poll changeCount, don't poll content

**Pitfalls to avoid:**
- Not filtering transient/concealed types (password managers)
- Polling content instead of changeCount
</context>

<task type="auto">
  <name>Task 1: Create ClipboardMonitor service</name>
  <files>clipass/Services/ClipboardMonitor.swift</files>
  <action>
Create clipass/Services/ClipboardMonitor.swift with:

1. @Observable class ClipboardMonitor
2. Properties:
   - history: [ClipboardItem] (published, starts empty)
   - private pasteboard: NSPasteboard.general
   - private lastChangeCount: Int
   - private timer: DispatchSourceTimer?

3. Ignored types array:
   - "org.nspasteboard.TransientType"
   - "org.nspasteboard.ConcealedType"
   - "org.nspasteboard.AutoGeneratedType"

4. Methods:
   - start(): Initialize timer with 500ms interval on background queue
   - stop(): Cancel timer
   - private poll():
     - Guard changeCount != lastChangeCount
     - Check for ignored types, return if found
     - Get string content, return if nil
     - Capture sourceApp via NSWorkspace.shared.frontmostApplication?.bundleIdentifier
     - Create ClipboardItem and insert at history[0] on main queue

**Avoid:**
- Reading clipboard content on every poll — only when changeCount changes
- Blocking main thread — use background DispatchQueue for polling
  </action>
  <verify>swift build succeeds</verify>
  <done>ClipboardMonitor compiles, has start/stop methods, filters ignored types</done>
